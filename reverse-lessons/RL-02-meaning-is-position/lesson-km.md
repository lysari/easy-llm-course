# មេរៀនបញ្ច្រាសទី ០២ — អត្ថន័យ គឺផ្អែកលើទីតាំង (Reverse Lesson 02 — Meaning Is Position)

---

## ស្រទាប់ដែលត្រូវបកស្រាយ៖ ពីចំនួនគត់ → ទៅជាវ៉ិចទ័រ (Layer to peel: integers → vectors)

យើងមានលេខសម្គាល់ថូខិន (Token IDs)៖ `[464, 3797, 3332, 319, 262, 2603]`

(ដែលតំណាងឱ្យឃ្លាថា៖ "the cat sat on the mat")

ជំហានបន្ទាប់របស់ម៉ូដែល (Model) គឺ៖ យកលេខសម្គាល់ (ID) នីមួយៗទៅរុករកក្នុង **តារាងបង្កប់ន័យ (Embedding table)** រួចជំនួសវាដោយ វ៉ិចទ័រ (Vector) ដែលជាសំណុំនៃលេខក្បៀស (Floating-point numbers)។

```
3797 ("cat")  →  [0.23, -0.81, 0.44, 0.12, -0.37, ...]
                  ↑ មានចំនួនរហូតដល់ 768 លេខក្នុង GPT-2 និងដល់ទៅ 12288 លេខក្នុង GPT-4

```

ជំហាននេះត្រូវបានគេហៅថា **ការបង្កប់ន័យ (Embedding)**។ វាជាជំហានដែលមនុស្សភាគច្រើនតែងតែគិតថា៖ "អូ! ពេលនេះហើយដែលអត្ថន័យត្រូវបានបញ្ចូលទៅ—ពេលនេះហើយដែលម៉ូដែលដឹងថា 'ឆ្មា (cat)' ជាអ្វី។"

តាមពិតមិនមែនទាល់តែសោះ។ តោះមកពិនិត្យមើលថាតើការបង្កប់ន័យ (Embedding) នេះជាអ្វីឱ្យប្រាកដ។

---

## តើការបង្កប់ន័យ (Embedding) ជាអ្វីឱ្យប្រាកដ?

ការបង្កប់ន័យ (Embedding) គ្រាន់តែជា **កូអរដោនេមួយនៅក្នុងលំហដែលមានច្រើនវិមាត្រ (Coordinate in a high-dimensional space)** តែប៉ុណ្ណោះ។

សូមស្រមៃគិតថាវាដូចជាកូអរដោនេ GPS ដែរ ប៉ុន្តែជំនួសឱ្យការមានត្រឹមតែ ២ វិមាត្រ (រដ្ឋបាលយ រយៈបណ្តោយ ឬ Latitude, Longitude) វាមានរហូតដល់ទៅ 768 វិមាត្រ។

```
ទីតាំងនៅលើផែនដី៖
  ទីក្រុងប៉ារីស (Paris) = (48.8566° N, 2.3522° E)      ← ប្រើលេខ ២ ខ្ទង់
  ទីក្រុងឡុងដ៍ (London) = (51.5074° N, 0.1278° W)     ← ប្រើលេខ ២ ខ្ទង់

ការបង្កប់ន័យពាក្យ (Word embedding)៖
  "cat"  = [0.23, -0.81, 0.44, ...]    ← ប្រើលេខ ៧៦៨ ខ្ទង់
  "dog"  = [0.19, -0.77, 0.51, ...]    ← ប្រើលេខ ៧៦៨ ខ្ទង់

```

ដូចគ្នានឹងកូអរដោនេ GPS ដែលមិនបានប្រាប់អ្នកថា ទីក្រុងប៉ារីស "ជាអ្វី" (វាគ្រាន់តែប្រាប់ថា ទីក្រុងប៉ារីស "នៅកន្លែងណា" លើផែនទី) នោះការបង្កប់ន័យ (Embeddings) ក៏មិនបានប្រាប់ម៉ូដែលថាពាក្យ "cat" មានន័យថាម៉េចដែរ។ វាគ្រាន់តែប្រាប់ថា "cat" ស្ថិតនៅត្រង់ចំណុចណាមួយនៅក្នុងលំហគណិតវិទ្យាដែលមាន 768 វិមាត្រប៉ុណ្ណោះ។

---

## តើការបង្កប់ន័យ (Embeddings) បានមកពីណា?

តារាងបង្កប់ន័យ (Embedding table) គឺត្រូវ **រៀន និងបង្កើតឡើងក្នុងអំឡុងពេលហ្វឹកហាត់ម៉ូដែល (Learned during training)**។

នៅពេលចាប់ផ្តើម៖ រាល់ទិន្នន័យបង្កប់ន័យ (Embedding) ទាំងអស់គ្រាន់តែជាលេខឥតប្រយោជន៍ និងរាយប៉ាយ—ពោលគឺវាជាលេខចៃដន្យ (Random numbers) តែប៉ុណ្ណោះ។

ក្នុងអំឡុងពេលហ្វឹកហាត់៖ លេខទាំងនោះត្រូវបានកែតម្រូវបន្តិចម្តងៗ (តាមរយៈវិធីសាស្ត្រ Gradient descent) រហូតទាល់តែថូខិន (Tokens) ណាដែលឧស្សាហ៍បង្ហាញខ្លួនក្នុងបរិបទ (Context) ស្រដៀងគ្នា រំកិលទៅនៅក្បែរគ្នានៅក្នុងលំហ 768 វិមាត្រនោះ។

```
ពេលហ្វឹកហាត់ ម៉ូដែលឃើញឃ្លា៖ "The cat sat" និង "The dog sat" រាប់លានដង
លទ្ធផល៖ កូអរដោនេនៃពាក្យ "cat" និង "dog" រំកិលទៅនៅជិតគ្នា
        ពីព្រោះវាបង្ហាញខ្លួនក្នុងទីតាំងស្រដៀងគ្នានៅក្នុងប្រយោគ។

```

**ម៉ូដែលមិនបានរៀនយល់ដឹងថា ឆ្មា (cats) និង ឆ្កែ (dogs) ជាសត្វធាតុនោះទេ។** វាគ្រាន់តែរៀនដឹងថា ពាក្យទាំងពីរនេះកើតឡើងនៅក្នុងទីតាំងប្រយោគស្រដៀងគ្នា។ ភាពនៅក្បែរគ្នានៅក្នុងលំហបង្កប់ន័យ (Embedding space) នេះគ្រាន់តែឆ្លុះបញ្ចាំងពី *ភាពស្រដៀងគ្នានៃការចែកចាយពាក្យ (Distributional similarity)* ប៉ុណ្ណោះ មិនមែនជាការយល់ដឹងពីអត្ថន័យពាក្យ (Semantic category) ឡើយ។

---

## ការតេស្តដ៏សំខាន់ដើម្បីផ្ទៀងផ្ទាត់ (The crucial test)

នេះជាវិធីដែលជួយឱ្យអ្នកដឹងច្បាស់ថា ការបង្កប់ន័យ (Embeddings) មិនបានផ្ទុកអត្ថន័យពិតប្រាកដនោះទេ៖

ប្រសិនបើអ្នកប្តូរពាក្យទាំងអស់នៅក្នុងទិន្នន័យហ្វឹកហាត់ (Training data) ជំនួសដោយកូដអក្សរចៃដន្យចំនួន ៦ ខ្ទង់វិញ៖

```
"cat" → "XQVBZM"
"dog" → "RFPKLA"
"sat" → "WNHQDS"

```

...ហើយយកទៅហ្វឹកហាត់ម៉ូដែលលើអត្ថបទដែលបានប្តូរនោះ កូដ "XQVBZM" និង "RFPKLA" នៅតែទទួលបានទិន្នន័យបង្កប់ន័យ (Embeddings) ស្រដៀងគ្នដដែល ពីព្រោះពួកវាបង្ហាញខ្លួនក្នុងទីតាំងប្រយោគស្រដៀងគ្នា។

ម៉ូដែលនឹងដំណើរការដូចមុនបេះបិទ។ វាជួយបង្កើតពាក្យ XQVBZM ជំនួសឱ្យពាក្យ cat ប៉ុន្តែ "កម្រិតឆ្លាតវៃ" របស់វានៅដដែល។

**ពាក្យពេចន៍ទាំងឡាយគ្រាន់តែជាស្លាកសញ្ញា (Labels)។ ចំណែកឯអត្ថន័យដែលផ្សារភ្ជាប់នឹងពាក្យទាំងនោះ គឺរស់នៅក្នុងខួរក្បាលរបស់អ្នក មិនមែននៅក្នុងម៉ូដែលនោះទេ។**

---

## អ្វីដែលវ៉ិចទ័រអាចធ្វើបាន (និងមិនអាចធ្វើបាន)

ការបង្កប់ន័យ (Embeddings) មានលក្ខណៈសម្បត្តិដ៏ល្បីមួយ គឺ៖ **ប្រមាណវិធីគណិតវិទ្យាលើវ៉ិចទ័រ (Vector arithmetic) បង្ហាញឱ្យឃើញពីទំនាក់ទំនងនៃអត្ថន័យពាក្យយ៉ាងស៊ីសង្វាក់គ្នា**។

```
embedding("king") - embedding("man") + embedding("woman")
≈ embedding("queen")

```

មើលទៅហាក់ដូចជាម៉ូដែលយល់ដឹងអំពី ខ្សែរាជវង្ស, យេនឌ័រ និងឋានានុក្រមនយោបាយអញ្ចឹង។

ប៉ុន្តែអ្វីដែលកើតឡើងពិតប្រាកដគឺ៖ ពាក្យដែលបង្ហាញខ្លួនក្នុងបរិបទស្រដៀងគ្នា ទទួលបានវ៉ិចទ័រស្រដៀងគ្នា។ ពាក្យ "King" និង "queen" បង្ហាញខ្លួនក្នុងបរិបទស្រដៀងគ្នា, ពាក្យ "king" និង "man" បង្ហាញខ្លួនក្នុងបរិបទស្រដៀងគ្នា — ហើយភាពខុសគ្នានៃវ៉ិចទ័រ (Vector differences) ក៏ត្រូវគ្នាចំពេលដោយចៃដន្យ។

វាគ្រាន់តែជា **ភាពចៃដន្យខាងធរណីមាត្រនៃការកើតឡើងរួមគ្នា (Geometric accident of co-occurrence)** ប៉ុណ្ណោះ មិនមែនជាការកត់ត្រាការពិតអំពីពិភពលោកនោះទេ។

ម៉ូដែលមិនមានទិន្នន័យណាមួយចែងថា៖ `king = ស្តេចជាបុរស` នោះឡើយ។ វាមានត្រឹមកូអរដោនេមួយ ហើយទីតាំងរបស់កូអរដោនេនោះធៀបនឹងកូអរដោនេដទៃទៀត បង្កើតបានជាលំនាំមួយ ដែលមនុស្សយើងយកមកបកស្រាយថាជា "អត្ថន័យ"។

---

## ការបង្កប់ន័យ (Embeddings) គ្រាន់តែជាចំណុចចាប់ផ្តើមប៉ុណ្ណោះ

បន្ទាប់ពីដំណើរការបំប្លែងជាថូខិន (Tokenization) រួច រាល់លេខសម្គាល់ថូខិន (Token ID) នីមួយៗក្លាយទៅជាវ៉ិចទ័រ។ បន្ទាប់មកវ៉ិចទ័រនោះត្រូវបាន៖

1. បូកបញ្ចូលជាមួយ ការកូដកម្មវិធីតាមទីតាំង (Positional encoding) ដើម្បីសម្គាល់លំដាប់លំដោយពាក្យ
2. បញ្ជូនឆ្លងកាត់ ស្រទាប់យកចិត្តទុកដាក់ (Attention layers)
3. បញ្ជូនឆ្លងកាត់ ស្រទាប់បញ្ជូនទៅមុខ (Feed-forward layers)
4. ហើយចុងក្រោយ បំប្លែងទៅជាប្រូបាប៊ីលីតេនៃលទ្ធផលចេញ (Output probabilities)

នៅគ្រប់ជំហានទាំងអស់៖ **មានតែលេខចូល និងលេខចេញតែប៉ុណ្ណោះ។**

តារាងបង្កប់ន័យ (Embedding table) មិនមែនជាវចនានុក្រមឡើយ។ វាគ្រាន់តែជាតារាងស្វែងរកកូអរដោនេ។ គ្មានអ្វីលើសពីនេះទេ។

```
ពាក្យដែលមានអត្ថន័យ     →   ចំនួនគត់ (Token IDs)
             ↓
ចំនួនគត់ដែលមានអត្ថន័យ  →   វ៉ិចទ័រកូអរដោនេ (លេខក្បៀស 768 ខ្ទង់)

```

ឥឡូវនេះយើងមានវ៉ិចទ័រហើយ។ ប៉ុន្តែវ៉ិចទ័រគ្រាន់តែជាសំណុំជួរនៃលេខ (Arrays of numbers)។ នៅតែមិនទាន់មានការយល់ដឹងដដែល។
