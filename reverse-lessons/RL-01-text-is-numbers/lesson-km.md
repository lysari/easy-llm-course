# Reverse Lesson 01 — អត្ថបទគឺជាលេខ (Text Is Numbers)

---

## ស្រទាប់ដែលត្រូវបកស្រាយ៖ ពាក្យ → ចំនួនគត់ (Words → Integers)

មេរៀនមុនបានបង្ហាញអ្នកពីការបំភាន់ភ្នែក។ ឥឡូវនេះ យើងនឹងបកស្រាយស្រទាប់ដំបូងបង្អស់។

អ្នកវាយបញ្ចូល៖ `"Hello, how are you?"`

មុនពេលការ "គិត" ណាមួយកើតឡើង ម៉ូដែល (Model) ធ្វើរឿងមួយមុនគេបង្អស់៖

**វាបំប្លែងអត្ថបទរបស់អ្នកទៅជាបញ្ជីនៃចំនួនគត់ (Integers)។**

```
"Hello, how are you?"
     ↓
[15496, 11, 703, 389, 345, 30]

```

ចំនួនគត់ទាំងនេះត្រូវបានគេហៅថា **ថូខិន (Tokens)**។ ដំណើរការនៃការបំប្លែងអត្ថបទទៅជាថូខិន ត្រូវបានគេហៅថា **ថូខិននីហ្សេសិន (Tokenization)**។

បន្ទាប់ពីជំហាននេះ **ម៉ូដែលមិនដែលឃើញពាក្យរបស់អ្នកម្តងទៀតឡើយ។** វាធ្វើការតែជាមួយលេខតែប៉ុណ្ណោះ។

---

## អ្វីទៅជាថូខិន (Token)?

ថូខិន គឺជាដុំកាត់នៃអត្ថបទ — ជាទូទៅវាអាចជាពាក្យមួយ ផ្នែកនៃពាក្យ ឬសញ្ញាវណ្ណយុត្តិ។ ថូខិននីមួយៗមានលេខសម្គាល់ (ID number) នៅក្នុងតារាងវាក្យសព្ទ (Vocabulary table) ដែលបានកំណត់ថេរមួយ។

ឧទាហរណ៍ (វាក្យសព្ទរបស់ GPT-2)៖

```
"the"    →  262
"cat"    →  3797
"sat"    →  3332
"on"     →  319
"mat"    →  2603
"."      →  13

```

ម៉ូដែលមានវាក្យសព្ទប្រហែល ~50,000 ថូខិន។ រាល់ថូខិនដែលអាចមានទាំងអស់ មានលេខសម្គាល់ (ID) ចាប់ពី 0 ដល់ 49,999។

---

## ហេតុអ្វីបានជាប្រើថូខិន ហើយមិនប្រើតួអក្សរ?

ថូខិន គឺធំជាងតួអក្សរទោល ប៉ុន្តែតូចជាងពាក្យពេញ។ នេះគឺជាជម្រើសផ្នែកវិស្វកម្មជាក់ស្តែង៖

* **ប្រើតែតួអក្សរ៖** ត្រូវឆ្លងកាត់ច្រើនជំហានពេកដើម្បីបង្កើតន័យ ធ្វើឱ្យដំណើរការយឺតខ្លាំង។
* **ប្រើពាក្យពេញ៖** មានពាក្យប្លែកៗច្រើនពេក (100,000+ ពាក្យ) ហើយពាក្យកម្រៗនឹងមិនទទួលបានទិន្នន័យសម្រាប់រៀន (No data)។
* **ប្រើផ្នែកនៃពាក្យ (Subword tokens)៖** ពាក្យទូទៅក្លាយជាថូខិនទោល ចំណែកពាក្យកម្រត្រូវបានបំបែកជាបំណែកតូចៗ។

```
"unhappiness"  →  ["un", "happiness"]  →  [403, 25763]
"tokenization" →  ["token", "ization"] →  [30001, 1634]

```

ពាក្យថា `"tokenization"` គឺមិនមានវត្តមានជាធាតុតែមួយក្នុងតារាងឡើយ។ ម៉ូដែលមើលឃើញវាជាពីរផ្នែកដាច់ដោយឡែកពីគ្នា។

---

## ចំណុចសំខាន់បំផុត (The critical point)

នៅខណៈពេលដែលធ្វើថូខិននីហ្សេសិន (Tokenization) **រាល់អត្ថន័យភាសាទាំងអស់ត្រូវបានលុបបំបាត់ចោល។**

ម៉ូដែលទទួលបាន៖ `[262, 3797, 3332, 319, 2603, 13]`

លេខទាំងប្រាំមួយខ្ទង់នោះមិនមានផ្ទុកខ្លឹមសារន័យវិទ្យា (Semantic content) ឡើយ។ ពួកវាគ្រាន់តែជាទីតាំងនៅក្នុងតារាងស្វែងរក (Lookup table) តែប៉ុណ្ណោះ។ លេខ `3797` មិនមែនមានន័យថា "ឆ្មា (cat)" នោះទេ។ វាមានន័យថា "ជួរដេកទី 3797 នៅក្នុងតារាងវាក្យសព្ទ"។

```
តារាងវាក្យសព្ទ (សម្រួលឱ្យងាយយល់)៖
  ជួរដេក 0:     "!"
  ជួរដេក 1:     " "
  ...
  ជួរដេក 262:   "the"
  ...
  ជួរដេក 3797:  "cat"    ← នេះជាអ្វីគ្រប់យ៉ាងដែល "cat" មានសម្រាប់ម៉ូដែល៖ គឺជួរដេក 3797
  ...
  ជួរដេក 49999: <ថូខិនចុងក្រោយ>

```

ម៉ូដែលមិនដឹងថាឆ្មាជាសត្វ មិនដឹងថាសត្វជា உயிரទម្រង់ (Living things) ឬក៏ உயிரទម្រង់ខុសពីវត្ថុគ្មានជីវិតនោះឡើយ។

---

## ថូខិននីហ្សេសិនមានការបាត់បង់ទិន្នន័យ (Tokenization is lossy)

Tokenization ក៏បោះបង់ចោលនូវ៖

* **អក្សរធំ-តូច** (ជួនកាល — អាស្រ័យលើ Tokenizer)
* **ព័ត៌មានលម្អិតនៃចន្លោះមិនឃើញ (Whitespace)** (ដូចជា Tabs ឬដកឃ្លាច្រើនដង)
* **អក្ខរាវិរុទ្ធដើម** នៃពាក្យកម្រ (ដោយសារពួកវាត្រូវបានបំបែកជាផ្នែកតូចៗ)

```
"GPT-4"   →  ["G", "PT", "-", "4"]  →  [38, 2767, 12, 19]

```

ម៉ូដែលមើលឃើញលេខបួនខ្ទង់នៅពេលអ្នកសរសេរ "GPT-4"។ ការផ្សារភ្ជាប់រវាងលេខទាំងបួននោះ ទៅនឹងគោលគំនិត "ម៉ូដែលភាសាដ៏ធំ (LLM) របស់ OpenAI" គឺមិនមានវត្តមាននៅក្នុងជំហាន Tokenization នេះឡើយ។

---

## ថូខិននីហ្សេសិនមិនមានការយល់ដឹងទេ

នេះជាសំណួរមួយដែលមើលទៅហាក់ដូចជាងាយស្រួល៖ **តើពាក្យមួយណាមានន័យវិជ្ជមានជាង?**

```
"happy" (សប្បាយចិត្ត)
"unhappy" (មិនសប្បាយចិត្ត)

```

មនុស្សអាចឆ្លើយបានភ្លាមៗថា៖ "happy" គឺវិជ្ជមាន ចំណែក "unhappy" គឺអវិជ្ជមាន។ បុព្វបទ (Prefix) "un-" គឺសម្រាប់បដិសេធ។

ឥឡូវមកមើលអ្វីដែល Tokenizer ធ្វើ៖

```
"happy"   →  [14774]
"unhappy" →  [403, 14774]    ← "un" + "happy"

```

ម៉ូដែលទទួលបានថូខិនពីរសម្រាប់ពាក្យ "unhappy"។ វាដាច់ខាតមិនបានទទួលសញ្ញាណ (Signal) ណាមួយដែលប្រាប់ថback ថូខិនទាំងនេះបដិសេធគ្នាទៅវិញទៅមកនោះឡើយ។ ទំនាក់ទំនងរវាង "un" (ការបដិសេធ) និង "happy" (អារម្មណ៍វិជ្ជមាន) គឺជាអ្វីដែលម៉ូដែលត្រូវរៀនសូត្រដោយខ្លួនឯងទាំងស្រុង ពីទម្រង់គំរូ (Patterns) នៅក្នុងអត្ថបទ — វាមិនត្រូវបានបង្កើតឡើងមកស្រាប់នៅក្នុងដំណើរការ Tokenization នោះទេ។

---

## អ្វីដែលម៉ូដែលទទួលបានជាធាតុចូល (Input)

បន្ទាប់ពីធ្វើ Tokenization រួច ប្រយោគបញ្ជា (Prompt) ទាំងស្រុងរបស់អ្នកនឹងក្លាយជាបញ្ជីនៃចំនួនគត់៖

```
"The cat sat on the mat" → [464, 3797, 3332, 319, 262, 2603]

```

នេះជាអ្វី **តែមួយគត់** ដែលចូលទៅក្នុងម៉ូដែល។ ចាប់ពីចំណុចនេះទៅ គឺគ្មានពាក្យ គ្មានប្រយោគ គ្មានច្បាប់វេយ្យាករណ៍ និងគ្មាននិយមន័យវចនានុក្រមឡើយ។

មានតែចំនួនគត់ប្រាំមួយខ្ទង់ប៉ុណ្ណោះ។

អ្វីៗគ្រប់យ៉ាងដែលម៉ូដែល "ដឹង" អំពីភាសា — មិនថាជាវេយ្យាករណ៍ អង្គហេតុ រចនាបថ ឬតក្កវិជ្ជា — ត្រូវតែលេចចេញមកពីទម្រង់គំរូ (Patterns) ដែលវាបានរៀន ក្នុងពេលដែលវាសម្លឹងមើលលំដាប់លំដោយនៃចំនួនគត់រាប់ពាន់លានបែបនេះ។

## ដំណើរការកូដសាកល្បង (Run the demo)

សូមមើល [demo.ts](demo.ts) — វាបង្កើត Tokenizer កម្រិតតួអក្សរ (Character-level tokenizer) យ៉ាងសាមញ្ញបំផុតមួយ ដើម្បីបង្ហាញពីរបៀបដែលអត្ថបទរលាយបាត់ទៅជាលេខ និងបង្ហាញថា Tokenizer មិនមានការយល់ដឹងទាល់តែសោះអំពីអ្វីដែលវាកំពុងធ្វើកូដនីយកម្ម (Encoding)។

## ស្ថានភាពរហូតមកដល់ពេលនេះ

```
អ្វីដែលអ្នកមើលឃើញ            អ្វីដែលជាក់ស្តែងមាននៅទីនោះ
──────────────        ──────────────────────
ពាក្យ, ប្រយោគ        →   បញ្ជីនៃចំនួនគត់ (Token IDs)
អត្ថន័យ              →   លេខជួរដេកនៅក្នុងតារាងស្វែងរក
"cat"             →   3797

```

ពាក្យសម្តីត្រូវបានរលាយបាត់អស់ហើយ។ យើងបានឈានចូលជ្រៅមួយស្រទាប់ទៀត ប៉ុន្តែវានៅតែមិនទាន់មាន "ការយល់ដឹង" ណាមួយកើតឡើងនៅឡើយទេ។
