# មេរៀនបញ្ច្រាសទី ០៣ — យន្តការយកចិត្តទុកដាក់ គឺជារូបមន្តគណិតវិទ្យា (Reverse Lesson 03 — Attention Is Arithmetic)



---

## ពិនិត្យមើលស្រទាប់ខាងក្នុង៖ ពីទំនាក់ទំនង ទៅជាផលគុណស្កាលែ (Layer to peel: relationships → dot products)



យើងមានលំដាប់នៃវ៉ិចទ័រ (Sequence of vectors) ដោយក្នុងមួយថូខិន (Token) មានវ៉ិចទ័រមួយ។

```
"The cat sat on the mat"
  ↓
[v_the, v_cat, v_sat, v_on, v_the, v_mat]
  វ៉ិចទ័រ v នីមួយៗ គឺជាបណ្តុំលេខចំនួន ៧៦៨ ខ្ទង់

```

បន្ទាប់មក **យន្តការយកចិត្តទុកដាក់ (Attention mechanism)** នឹងចាប់ផ្តើមដំណើរការ។ នេះគឺជាបេះដូងរបស់ម៉ូដែល Transformer។ ឈ្មោះថា "Attention" (ការយកចិត្តទុកដាក់) មើលទៅដូចជាម៉ូដែលនេះកំពុងផ្ដោតអារម្មណ៍ទៅលើអត្ថន័យ — ពោលគឺយល់ថាពាក្យណាទាក់ទងនឹងពាក្យណា។

ប៉ុន្តែតាមពិត វាមិនមែនបែបនោះទេ។

---

## អ្វីដែលយន្តការ Attention ធ្វើ៖ ការពន្យល់ក្នុងមួយប្រយោគ (What attention does: one sentence)



សូមពិចារណាប្រយោគនេះ៖ `"The animal didn't cross the street because it was too tired."` (សត្វនោះមិនបានឆ្លងថ្នល់ទេ ព្រោះវាហត់ខ្លាំងណាស់។)

តើពាក្យថា "it" (វា) សំដៅលើអ្វី? គឺសំដៅលើ "The animal" (សត្វនោះ) — មិនមែន "the street" (ថ្នល់) ទេ។

មនុស្សយើងពេលអានប្រយោគនេះនឹងដឹងភ្លាមថា៖

* សត្វចេះហត់


* ថ្នល់មិនចេះហត់ទេ


* ហេតុដូចនេះ "it" = "the animal"



ម៉ូដែល Transformer ចាត់ចែងពាក្យ "it" នេះដោយប្រើប្រាស់យន្តការ Attention។ ប៉ុន្តែ ខាងក្រោមនេះជាអ្វីដែលយន្តការ Attention គណនាជាក់ស្តែង៖

---

## ម៉ាទ្រីសទាំងបី៖ Q, K, V (The three matrices: Q, K, V)



វ៉ិចទ័រថូខិន (Token vector) នីមួយៗ ត្រូវយកទៅគុណនឹងម៉ាទ្រីសមេគុណទម្ងន់ (Weight matrices) ចំនួនបី ដែលម៉ូដែលបានរៀនសូត្រ៖

```
query  = token_vector × W_Q      (តើខ្ញុំកំពុងស្វែងរកអ្វី?)
key    = token_vector × W_K      (តើខ្ញុំមានព័ត៌មានអ្វីផ្តល់ឱ្យ?)
value  = token_vector × W_V      (តើខ្ញុំផ្ទុកព័ត៌មានអ្វីខ្លះ?)

```

ទាំងនេះគ្រាន់តែជាការគុណម៉ាទ្រីស (Matrix multiplications) ធម្មតា ដែលជាគណិតវិទ្យាលីនេអ៊ែរ (Linear algebra) កម្រិតស្តង់ដារ។ បញ្ចូលលេខទៅ បានលទ្ធផលជាលេខមកវិញ។

បន្ទាប់មក៖ សម្រាប់ថូខិននីមួយៗ ម៉ូដែលនឹងគណនាថាតើវាគួរ "ផ្តល់តម្លៃ/យកចិត្តទុកដាក់" ទៅលើថូខិនដទៃទៀតកម្រិតណា៖

```
attention_score(i, j) = dot_product(query_i, key_j)
                        ÷ sqrt(dimension)

```

ផលគុណស្កាលែ (Dot product) គឺសាមញ្ញណាស់៖ គ្រាន់តែយកលេខតាមទីតាំងត្រូវគ្នាមកគុណគ្នា រួចបូកលទ្ធផលចូលគ្នាទាំងអស់។

```
query_i = [0.3, -0.2, 0.8, 0.1]
key_j   = [0.5,  0.4, 0.7, 0.2]

dot product = (0.3)(0.5) + (-0.2)(0.4) + (0.8)(0.7) + (0.1)(0.2)
            = 0.15 - 0.08 + 0.56 + 0.02
            = 0.65

```

លទ្ធផលទទួលបានលេខមួយខ្ទង់។ នោះហើយជា "ពិន្ទុយកចិត្តទុកដាក់" (Attention score) រវាងថូខិន i និងថូខិន j។

---

## ប្រើប្រាស់ Softmax ដើម្បីទាញរកទម្ងន់យកចិត្តទុកដាក់ (Softmax to get attention weights)



ពិន្ទុទាំងឡាយត្រូវបានបញ្ជូនឆ្លងកាត់មុខងារ Softmax ដើម្បីបំប្លែងទៅជាប្រូបាប៊ីលីតេ (ទម្ងន់យកចិត្តទុកដាក់ - Attention weights)៖

```
scores  = [0.65, 0.20, 0.80, 0.10]   ← ផលគុណស្កាលែដើម (Raw dot products)
weights = softmax(scores)
        = [0.25, 0.15, 0.45, 0.10]   ← ផលបូកស្មើនឹង ១.០ ជានិច្ច

```

ថូខិនណាដែលមានពិន្ទុខ្ពស់ជាងគេ នឹងទទួលបាន "ការយកចិត្តទុកដាក់" ច្រើនជាងគេ។

---

## ការផ្សំបញ្ចូលគ្នាជាមួយតម្លៃ Value (Combining with values)



ជាចុងក្រោយ លទ្ធផល (Output) សម្រាប់ថូខិននីមួយៗ គឺជា **ផលបូកទម្ងន់នៃវ៉ិចទើតម្លៃ (Weighted sum of value vectors)**៖

```
output_i = sum(weight_j × value_j)  សម្រាប់ j ទាំងអស់

         = 0.25 × value_0
         + 0.15 × value_1
         + 0.45 × value_2      ← ថូខិននេះទទួលបានទម្ងន់ខ្លាំងជាងគេ
         + 0.10 × value_3

```

នេះគឺជាលទ្ធផលចេញពីក្បាលយកចិត្តទុកដាក់មួយ (Attention head) សម្រាប់ថូខិន i៖ វាជាមធ្យមភាគទម្ងន់ (Weighted average) នៃវ៉ិចទើតម្លៃរបស់ថូខិនផ្សេងៗទៀត។

គ្មានការយល់ដឹងអ្វីទាំងអស់។ វាគ្រាន់តែជាការរកមធ្យមភាគទម្ងន់ប៉ុណ្ណោះ។

---

## ត្រឡប់មកពាក្យ "it" វិញ (Back to "it")



ចុះហេតុអ្វីបានជាម៉ូដែលអាចដោះស្រាយរកន័យចំត្រឹមត្រូវថា "it" = "the animal" បាន?

ក្នុងអំឡុងពេលហ្វឹកហាត់ (Training) មេគុណទម្ងន់នៅក្នុងម៉ាទ្រីស W_Q, W_K, W_V ត្រូវបានកែតម្រូវតាមរយៈវិធីសាស្ត្រ Gradient descent រហូតទាល់តែសំណួរ (Query) របស់ពាក្យ "it" រត់ទៅរកគន្លឹះ (Key) របស់ពាក្យ "animal" បានល្អជាងពាក្យ "street" — ពីព្រោះនៅក្នុងប្រយោគរាប់លានដែលមានឃ្លា "it was tired" ពាក្យបរិបទជុំវិញគឺមានលក្ខណៈស្រដៀងនឹងសត្វ (Animal) ជាងថ្នល់ (Street)។

ម៉ូដែលមិនបានដឹងថា សត្វចេះហត់ ហើយថ្នល់មិនចេះហត់នោះទេ។ វាគ្រាន់តែរៀនសូត្រដឹងថានិមិត្តសញ្ញាវ៉ិចទ័រសម្រាប់ពាក្យ "animal" និង "tired" តែងតែមាននិន្នាការលេចឡើងរួមគ្នាក្នុងទម្រង់គំរូ (Patterns) ដែលមានពាក្យ "it" ជាមួយដែរ។ ដូចនេះ ផលគុណស្កាលែ (Query-key dot products) សម្រាប់វ៉ិចទ័រទាំងនោះក៏មានតម្លៃខ្ពស់ជាង។

**ក្របខ័ណ្ឌវេយ្យាករណ៍ គឺជារូបធរណីមាត្រ។ ចំណែកឯការយល់ដឹង គឺគ្រាន់តែជាស្ថិតិ។ (Grammar is geometry. Understanding is statistics.)**

## អ្វីដែលយន្តការ Attention អាចធ្វើបាន និងមិនអាចធ្វើបាន (What attention can and cannot do)



យន្តការ Attention អាច៖

* បញ្ជូនព័ត៌មានពីទីតាំងថូខិនមួយទៅទីតាំងថូខិនមួយទៀត


* ផ្តល់ទម្ងន់ដល់ថូខិនខុសៗគ្នា ផ្អែកលើធរណីមាត្រវ៉ិចទ័រដែលបានរៀនសូត្រ


* បង្កើតទម្រង់តំណាងដ៏ស្មុគស្មាញ (Complex representations) តាមរយៈស្រទាប់ Attention ជាច្រើនជាន់



យន្តការ Attention មិនអាច៖

* ស្វែងរកការពិតចេញពីមូលដ្ឋានទិន្នន័យចំណេះដឹង (Knowledge base)


* ផ្ទៀងផ្ទាត់ថាតើរបាយការណ៍ណាមួយជាការពិតឬអត់


* ពិចារណាគិតវែកញែកមួយជំហានម្តងៗបានឡើយ (បើគ្មានការហ្វឹកហាត់ពិសេស)


* ដឹងថាពាក្យមានន័យដូចម្តេច ក្រៅពីទម្រង់នៃការលេចឡើងរួមគ្នារបស់ពួកវា (Co-occurrence patterns)
